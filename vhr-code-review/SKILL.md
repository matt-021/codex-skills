# VHR 代码审查

VHR 项目统一代码审查 skill。支持详细审查报告和快速检查清单。

## 前置约束（强制）

- 注释必须详细，所有方法必须有注释。
  - Java 方法必须有 Javadoc。
  - 前端导出函数/组件必须有 JSDoc。
  - 复杂逻辑必须补充行内注释，说明“为什么这样做”。
- 所有文件必须使用 UTF-8 编码，禁止出现中文乱码；提交前需确认中文显示正常。

## 何时使用

- 审查 Pull Request 或 Merge Request
- 提交前检查代码变更
- 用户请求代码审查
- 检查代码是否符合 VHR 规范

## 审查格式

### 格式一：详细审查（推荐用于 PR/正式审查）

```markdown
# [模块/功能名称] 代码审查报告

## 结论

总体结论：**通过 / 需修改后合并 / 建议优化**

主要风险摘要（1-3 句话概述关键问题）

## 功能链路

* 前端页面：`path/to/page`
* 前端 API：`api.method.name`
* 后端 Controller：`ControllerClass.java`
* 后端 Service：`ServiceImpl.java`
* MyBatis SQL：`Mapper.xml`

## 阻断问题（必须修改）

### 1) [问题标题]

* 位置：`File.java:L10-L20`
* 问题点：描述具体问题及风险
* 建议：具体修复方案

## 重要问题（建议尽快）

### 2) [问题标题]

* 位置：`File.java:L30-L40`
* 问题点：描述
* 建议：修复方案

## 建议优化（可选）

### 3) [问题标题]

* 位置：`File.java:L50`
* 建议：优化方向

## 权限与安全审查

* 鉴权：是否使用正确的 @PermissionCheck
* 数据权限：是否绕过拦截器
* 入参校验：DTO 字段校验是否完整
* 敏感数据：是否有日志泄露风险
```

### 格式二：快速检查清单

```markdown
## 代码审查清单 - [模块名]

### 阻断项
- [ ] 无安全漏洞（SQL注入、XSS、越权）
- [ ] 入参使用 DTO（非 VO/Entity）
- [ ] 写操作使用 BaseService 方法
- [ ] 无硬编码配置/URL

### 重要项
- [ ] 公共方法有注释（Javadoc/JSDoc）
- [ ] 异常处理正确（抛出非返回错误字符串）
- [ ] 错误码规范（带服务前缀）
- [ ] 新增业务枚举实现 `LabelEnum` 且提供 `label`
- [ ] 分页使用 PageUtil.buildPage
- [ ] 前端新增按钮/标题/提示文案已接入 i18n（禁止硬编码中文文案）
- [ ] 前端优先复用 `ui-modules` 公共组件/公共 hooks（禁止重复造轮子）
- [ ] 前端请求统一走公共 axios 封装（禁止页面内直接 new axios）
- [ ] 鉴权请求确认携带 `Authorization: Bearer <token>`

### 建议项
- [ ] 无 N+1 查询
- [ ] 批量操作非循环单条
- [ ] 日志级别合理
```

## 审查工作流

```
1. 确定范围
   ├─ 仅前端 → vhr-fe-toolkit checklist
   ├─ 仅后端 → vhr-be-toolkit checklist
   └─ 全栈 → 两个 toolkit 都用

2. 读取变更文件
   ├─ Controller/Service/Mapper
   ├─ Vue components/hooks
   └─ DTO/VO/Entity

3. 对照规范检查
   ├─ 安全与权限
   ├─ 代码质量
   └─ 项目约定

4. 生成报告
   ├─ 详细版：完整报告含严重级别
   └─ 快速版：检查清单含通过/失败
```

## 分类常见问题

### 安全问题（阻断）

| 问题 | 模式 | 风险 |
|------|------|------|
| VO 作为入参 | `create(XxxVo vo)` | 越权篡改审计字段 |
| 越权操作 | 无 `@PermissionCheck(PERMISSION)` | 任意用户可执行 |
| 绕过拦截器 | `AdmThreadLocalUtil.addInterceptExclude()` | 数据权限失效 |
| 硬编码凭据 | URL/密码在代码中 | 安全泄露 |

### 正确性问题（阻断/重要）

| 问题 | 模式 | 风险 |
|------|------|------|
| 分页错误 | `Page.of()` 非 `PageUtil.buildPage()` | whereItems 失效 |
| NPE 风险 | 未判空直接调用 | 运行时异常 |
| 并发冲突 | `selectCount()` 生成编码 | 重复编码 |
| 条件 bug | `if test="enable"` 非 `enable != null` | false 值被忽略 |

### 质量问题（建议）

| 问题 | 模式 | 建议 |
|------|------|------|
| N+1 查询 | 循环内查询 | 批量查询回填 |
| 循环更新 | `for { update }` | 批量更新 |
| 魔法值 | `if (type == 1)` | 使用常量/枚举 |
| 重复代码 | 相同逻辑多处 | 抽取公共方法 |

### 国际化问题（阻断/重要）

| 问题 | 模式 | 风险 |
|------|------|------|
| 新增按钮文案硬编码 | `<vhr-button>审批回调记录</vhr-button>` | 多语言环境显示错误、无法统一维护 |
| 文案未落 i18n | 仅改页面未改 `src/lang/*` | 发版后不同语言不一致 |

### 认证链路一致性问题（阻断/重要）

| 问题 | 模式 | 风险 |
|------|------|------|
| 登录/登出绕过统一入口 | 页面直接拼接非标准地址调用认证接口 | 环境切换失败、链路不一致 |
| token 注入方式不统一 | 仅传 `access_token` 参数，未带 `Authorization` 头 | 网关鉴权失败（401/403） |
| 内部接口误走外部路径 | 缺少内部调用标识或调用方式错误 | 被网关拒绝或权限失效 |

### 公共组件复用问题（重要/建议）

| 问题 | 模式 | 风险 |
|------|------|------|
| 未复用 ui-modules | 已有组件场景重复自建实现 | 维护成本高、风格不一致 |
| 绕过公共 hooks | 页面手写重复分页/筛选/权限逻辑 | 易引入参数与行为偏差 |

## 严重级别

| 级别 | 图标 | 含义 | 处理 |
|------|------|------|------|
| 阻断 | 🔴 | 必须修改 | 不通过 |
| 重要 | 🟡 | 建议尽快修改 | 可通过但有风险 |
| 建议 | 🟢 | 建议优化 | 可忽略 |
